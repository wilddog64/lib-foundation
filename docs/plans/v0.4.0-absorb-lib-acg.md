# v0.4.0 — Absorb lib-acg into lib-foundation as an optional `acg/` module

**Goal:** Retire the standalone `lib-acg` repo and the 3-level subtree chain
(foundation → lib-acg → k3d-manager). Move ACG automation into lib-foundation as an **optional
module** at `scripts/lib/acg/` that sources core (`_run_command`, logging) by relative path — one
authored copy of `_run_command`, no vendoring, portable to any foundation consumer.

**Repos touched:** lib-acg (source + retire), lib-foundation (new home), k3d-manager (rewire).

**Why:** `acg.sh` calls `_run_command` 21×; it cannot be inlined without forking the function that
drives k3d-manager's privilege model. The dependency is correct (acg → foundation); only the
*mechanism* (full-repo subtree vendor) is the overhead. Co-locating in one repo gives single source
of truth + portability + collapses k3d-manager from **two** vendored subtrees to **one**.

---

## Target layout in lib-foundation

```
scripts/lib/system.sh              # core — source alone = ZERO node deps (unchanged)
scripts/lib/acg/                   # NEW optional module
  cdp.sh                           #   sources ../system.sh (guarded) — one copy of _run_command
  acg.sh        gcp.sh             #   former scripts/plugins/*  (the public acg_*/gcp_* API)
  vars.sh
  acg_session_check.js
  playwright/                      #   acg_restart.js, acg_credentials.js, providers/, lib/
  etc/acg-cluster.yaml             #   former scripts/etc/
  bin/                             #   acg-credential-test, acg-extend-test
  package.json  package-lock.json  #   node deps live HERE — only pulled if you use acg
  tests/                           #   acg bats
```

Core stays minimal to *load*; ACG is opt-in. Sourcing `scripts/lib/system.sh` never touches node.

**Rewire rule:** every acg `.sh` that uses foundation helpers gets a guarded bootstrap at top:
```bash
if ! declare -f _run_command >/dev/null 2>&1; then
  # shellcheck source=/dev/null
  source "$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)/system.sh"
fi
```
This makes the module work standalone (other projects) AND when the host already loaded foundation
(k3d-manager) — the `declare -f` guard means the host's copy always wins.

**Do NOT migrate:** lib-acg's `docs/`, `memory-bank/`, `retro/`, `issues/`. They stay in the
archived lib-acg repo. Bring only a single `docs/api/acg.md` summary into foundation.

## Phase 1 — Import + rewire in lib-foundation (PR on `feat/v0.4.0`)

1. **Import runtime tree** from lib-acg `main` into `scripts/lib/acg/` as a **clean copy** (single
   commit; history stays recoverable in the archived lib-acg repo — no subtree mechanics):
   ```bash
   # from a clean lib-acg main checkout, copy runtime paths into foundation/scripts/lib/acg/
   #   scripts/lib/cdp.sh, scripts/lib/acg_session_check.js
   #   scripts/plugins/acg.sh -> acg/acg.sh ; scripts/plugins/gcp.sh -> acg/gcp.sh
   #   scripts/vars.sh, scripts/etc/, bin/, playwright/, package*.json, tests (acg bats)
   ```
   Do NOT copy lib-acg's vendored `scripts/lib/foundation/` (that's the whole point).

2. **Rewire helper resolution** — add the guarded `../system.sh` bootstrap (above) to `cdp.sh`,
   `acg.sh`, `gcp.sh`. Remove cdp.sh's old vendored-foundation fallback block. Fix any
   `_LIB_ACG_ROOT` / relative paths that assumed the old layout (now rooted at `scripts/lib/acg/`).

3. **CI** — add a separate `acg` job to `.github/workflows/ci.yaml`: node setup → `npm ci` →
   `node --check` on the JS → acg bats. Keep it **independent** of the core shellcheck job so a
   Playwright/node flake never blocks a core foundation change. The live `make credential-test`
   gate stays manual (needs a real sandbox + browser) — not in CI.

4. **Docs** — `docs/api/acg.md` summary; CHANGELOG `[0.4.0]` Added (acg module) / note the source
   lib-acg SHA. Update `README.md`, `CLAUDE.md`, `.github/copilot-instructions.md` to list the
   new module + its public `acg_*` / `gcp_*` API and node/playwright review rules.

5. **Verify:** `bash -n` + `shellcheck -S warning` on all acg `.sh`; `node --check` on JS;
   `source scripts/lib/acg/acg.sh` standalone defines `acg_*` with `_run_command` resolved from
   `../system.sh`; core still sources clean with zero node.
