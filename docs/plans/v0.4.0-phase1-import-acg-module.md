## Goal

Move the lib-acg runtime tree into lib-foundation under a single **optional** module `scripts/lib/acg/`
that resolves foundation helpers via `../system.sh` (one authored `_run_command`, no vendoring).
Sourcing core (`scripts/lib/system.sh`) must still touch **zero** node deps — acg is opt-in.

The whole point: drop lib-acg's vendored `scripts/lib/foundation/` (the 3-level subtree chain) and
collapse to one source of truth.

## Change 2 — Rewire `scripts/lib/acg/cdp.sh`

The module now lives at `scripts/lib/acg/`, so `../system.sh` resolves to `scripts/lib/system.sh`,
and `_LIB_ACG_ROOT` is the module dir itself (not `../..`).

**Exact old block (cdp.sh lines 8–17):**

```bash
# Ensure _antigravity_browser_ready is available (provided by lib-foundation/system.sh)
if ! declare -f _antigravity_browser_ready >/dev/null 2>&1; then
  _CDP_FOUNDATION="$(cd "$(dirname "${BASH_SOURCE[0]}")/foundation/scripts/lib" && pwd)"
  # shellcheck source=/dev/null
  [[ -f "${_CDP_FOUNDATION}/system.sh" ]] && source "${_CDP_FOUNDATION}/system.sh"
fi

if [[ -z "${_LIB_ACG_ROOT:-}" ]]; then
  _LIB_ACG_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)"
fi
```

**Exact new block:**

```bash
# Resolve foundation helpers. When a host already loaded foundation (k3d-manager),
# the host copy wins via the declare -f guard. Standalone, pull ../system.sh —
# this module lives at scripts/lib/acg/, so ../ is scripts/lib/.
if ! declare -f _run_command >/dev/null 2>&1; then
  # shellcheck source=/dev/null
  source "$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)/system.sh"
fi

if [[ -z "${_LIB_ACG_ROOT:-}" ]]; then
  _LIB_ACG_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
fi
```

**Then** fix the one flattened path (cdp.sh line 105):

```bash
  local _acg_session_check_script="${_LIB_ACG_ROOT}/scripts/lib/acg_session_check.js"
```
→
```bash
  local _acg_session_check_script="${_LIB_ACG_ROOT}/acg_session_check.js"
```

The `${_LIB_ACG_ROOT}/node_modules` references (lines 112, 113, 117) stay unchanged — `node_modules`
installs under the module dir.

## Change 6 — Add the independent `acg` CI job

**EDIT `.github/workflows/ci.yaml`** — append this job after the existing `bats` job (same indentation
level under `jobs:`). Keep it independent so a node/Playwright flake never blocks a core change.

```yaml
  acg:
    name: acg (node)
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: scripts/lib/acg
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install deps (npm ci from lockfile)
        run: npm ci

      - name: Node syntax check
        run: npm run check

      - name: Unit tests (jest)
        run: npm test

      - name: shellcheck bin scripts
        run: |
          sudo apt-get install -y shellcheck
          shellcheck -S warning bin/acg-credential-test bin/acg-extend-test
```

> The Playwright e2e (`tests/acg-restart.spec.js`, `npm run test:e2e`) needs a real browser — it
> stays a manual gate (`make credential-test`), NOT in CI. Only the jest unit tests run here.
> The core `shellcheck` job already globs `scripts/lib/**/*.sh`, so it will pick up
> `scripts/lib/acg/{cdp,acg,gcp}.sh` automatically — they MUST be clean at that job's default
> severity (see Rules).
