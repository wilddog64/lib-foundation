# Retrospective — v0.4.1

**Date:** 2026-07-06
**Milestone:** Headless Pluralsight auto-login + non-interactive fast-fail for unattended AWS-sandbox provisioning
**PR:** #33 — merged to main (`b7c849c`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- **Upstream-first discipline held.** The feature was implemented and shipped in lib-foundation
  (`feat/v0.4.1`) and the k3d-manager subtree-pull was deliberately deferred until v0.4.1 landed via
  tag — the ACG code has exactly one source of truth, and k3d-manager `scripts/lib/foundation/**`
  stayed untouched until the release existed to pull from.
- **Codex's implementation was independently verified, not trusted.** Every "done" was checked: SHAs
  present on `origin/feat/v0.4.1`, correct remote, all changes scoped under `scripts/lib/acg/`, secret
  hygiene confirmed (creds passed as `ACG_USERNAME`/`ACG_PASSWORD` env vars, never on argv), and the
  `_secret_load_data <service> <key>` argument order verified against foundation `system.sh`.
- **Fast-fail contract closed a real unattended-provisioning hole.** A logged-out or half-filled CDP
  Chrome session used to leave `acg_session_check.js` polling five minutes for a human, making a
  Slack-launched `/cluster-up aws` unrecoverable. It now auto-logs-in when creds exist and fails fast
  (`ACG_SESSION_EXPIRED`, no poll) under `K3DM_NONINTERACTIVE=1` / no TTY, and refuses MFA rather than
  hanging.
- **The declined Copilot finding validated "verify before trust" against an automated reviewer.**
  Copilot flagged `browser.close()` as inconsistent and wanted `browser.disconnect()`. Verified
  against the vendored Playwright 1.60.0 type defs that `Browser.disconnect()` does not exist
  (`disconnected` is only an event); `close()` is the documented CDP-safe teardown. Declined with
  evidence — and surfaced that `gcp_login.js:157`'s `disconnect()` is a latent no-op.

## What Went Wrong

- **CHANGE.md `[Unreleased]` was missing the feature entry** and had to be added during create-pr
  pre-flight — the same versioning drift v0.4.0 hit (it merged unstamped). The changelog is still not
  written as part of the implementation commit.
- **`gcp_login.js:157` carries a latent no-op `disconnect()`** whose `TypeError` is swallowed by an
  empty `catch`. Discovered via Copilot finding #2; deferred to a separate follow-up rather than
  fixed opportunistically in this PR.
- **The initial non-interactive tests were near-duplicates** (both no-creds) and neither exercised the
  `_autoLogin` MFA-refuse path — Copilot finding #3. The spec's coverage list didn't distinguish the
  MFA branch from the no-creds branch.

## Process Rules Added

| Rule | Where | Why |
|------|-------|-----|
| Verify a Copilot "consistency" suggestion against the vendored dependency's type defs before changing working code — a suggestion that matches a sibling file is not evidence the sibling is correct | docs/issues/2026-07-05-copilot-pr33-review-findings.md | Finding #2: `browser.disconnect()` does not exist in Playwright 1.60.0; the sibling `gcp_login.js` was itself wrong |

## Decisions Made

- **Upstream-first, then subtree-pull.** Land in lib-foundation → tag v0.4.1 → subtree-pull into
  k3d-manager. No parallel copy in k3d-manager while the feature is in flight.
- **`browser.close()` is the CDP teardown, not `disconnect()`.** For a `connectOverCDP` browser,
  `close()` clears Playwright-created contexts and detaches without terminating the operator's Chrome.
- **Fast-fail over human-poll when non-interactive.** `K3DM_NONINTERACTIVE=1` / no TTY → immediate
  `ACG_SESSION_EXPIRED`; MFA challenge → refuse (`mfa_required`), never attempt to solve.
- **Follow-up (deferred):** correct the misleading `disconnect()` idiom in `gcp_login.js`.

## Theme

v0.4.1 makes the ACG Pluralsight session usable unattended: it logs itself in when credentials are
available and, when they are not, fails fast instead of blocking a Slack-launched provision on a
human who will never arrive. The milestone's discipline was in the verification, not the code — a
22-step Codex implementation checked line by line against the spec, credentials kept off argv, and a
plausible-sounding Copilot suggestion correctly declined only after reading the vendored Playwright
type definitions. The recurring soft spot remains changelog stamping, which still lands at PR time
rather than in the implementation commit.
