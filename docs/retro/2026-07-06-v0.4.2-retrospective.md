# Retrospective — v0.4.2

**Date:** 2026-07-06
**Milestone:** Headless CDP auto-login with stale-browser reclaim/reuse on the AWS-sandbox credential-test path
**PR:** #34 — merged to main (`ae9fc73`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- **The v0.4.1 headless gate was actually wired into production this milestone.** v0.4.1 shipped
  `_cdp_ensure_acg_session` / `acg_session_check.js` / `loginWithPage` but no production path called
  them, so `make up` still relied on Google-PW autofill that cannot fire in the `--password-store=basic`
  dedicated profile — sign-in silently no-op'd and AWS provisioning died on CFN `InvalidClientTokenId`.
  BUG #1 wired the gate into both `_browser_launch` paths and BUG #2 into `bin/acg-credential-test`, so
  both the `make up` and `make credential-test` flows now enforce a live Pluralsight session.
- **The browser-identity problem was solved by owning the browser, not guessing about it.** BUG #3
  switched CDP from system Chrome to Playwright-managed Chromium (`chromium.executablePath()`, hard-fail
  with `npm install` guidance, no system-Chrome fallback), and BUG #5 replaced the BUG #4 hard-error
  reuse branch with a real health probe: `_cdp_connectable` drives a non-destructive `connectOverCDP`,
  reuse only when Playwright can actually drive the browser, otherwise `_cdp_kill_port_listener`
  reclaims `:9222` and relaunches. The path went from "error if anything is on the port" to
  "reuse healthy, reclaim stale."
- **Upstream-first held again.** All five BUGs landed in lib-foundation `feat/v0.4.2`; k3d-manager
  `scripts/lib/foundation/**` was left untouched until v0.4.2 was tagged, then subtree-pulled
  (`c116e655` / `b6141505`). One source of truth, verified diff-identical after the pull.
- **The operator's live gate is the real acceptance test.** Every BUG carried "acceptance remains
  user-run: `make credential-test PROVIDER=aws`" — the milestone did not claim done on shellcheck +
  BATS + sourcing probes alone. The user ran the live gate and confirmed headless operation before merge.

## What Went Wrong

- **A five-bug milestone to get one flow working is a design-debt signal.** BUG #1→#5 were iterative
  discoveries on the same seam (which browser, which profile, which port, reuse-vs-reclaim). BUG #4
  introduced a hard error that BUG #5 then had to replace — the reuse contract was not thought through
  before the first fix landed.
- **CI severity gap caught SC2016 that local shellcheck missed.** Local runs use `shellcheck -S warning`,
  which excludes `info`; lib-foundation CI runs default severity and flagged SC2016 on the
  `_cdp_connectable` node `-e` literal. Fixed test-time in `9be232a` with a `# shellcheck disable=SC2016`
  directive, but the first CI run failed for a class of finding the local gate cannot see.
- **BATS drifted behind the implementation across BUGs.** `acg_cdp.bats` `_browser_launch` tests went
  stale — test 2 drifted at BUG #3, test 1 at BUG #5 — and were only caught on the first PR CI run,
  fixed test-only in `d9ff95e` (mock `_cdp_connectable`, stub `node`, add reclaim-branch test). Tests
  were updated reactively at PR time, not alongside each behavioral change.
- **`gcp_login.js:157` latent no-op `disconnect()` is still open** — carried forward from v0.4.1,
  still deferred to a separate change.

## Process Rules Added

| Rule | Where | Why |
|------|-------|-----|
| Run default-severity shellcheck (not `-S warning`) before pushing to lib-foundation — CI runs default severity and will catch `info`-level findings (e.g. SC2016) the local `warning` gate hides | this retro | First PR #34 CI run failed on SC2016 that local `shellcheck -S warning` excluded |
| When a fix changes `_browser_launch` / CDP behavior, update `acg_cdp.bats` in the SAME commit — do not let the mock/stub set drift until PR CI catches it | this retro | `acg_cdp.bats` tests 1 and 2 drifted across BUG #3/#5 and were only caught on PR CI |

## Decisions Made

- **Own the browser: Playwright-managed Chromium, not system Chrome.** Resolve via
  `chromium.executablePath()`, hard-fail if absent, no system-Chrome fallback. Removes the
  version-mismatch and profile-lock ambiguity that made reuse-vs-relaunch unpredictable.
- **Reuse healthy, reclaim stale — never hard-error on a live port.** `_cdp_connectable` is the
  arbiter: a non-destructive `connectOverCDP().close()` probe decides reuse; failure triggers
  `lsof -t` port reclaim and a managed relaunch. Supersedes the BUG #4 hard-error branch.
- **The headless session gate must be called on every production sign-in path.** Both
  `_browser_launch` and `bin/acg-credential-test` invoke `_cdp_ensure_acg_session`; a shipped gate
  that nothing calls is not shipped.
- **Follow-up (still deferred):** correct the misleading no-op `disconnect()` in `gcp_login.js:157`.

## Theme

v0.4.2 finished the job v0.4.1 started: it took a headless-login gate that existed but was never
wired in, connected it to both production paths, and then spent four more bug fixes making the CDP
browser itself predictable — a Playwright-managed Chromium the tooling owns, a health probe that
decides reuse honestly, and a port-reclaim path that recovers from a stale or zombie browser instead
of erroring out. The engineering worked; the shape of the work was the lesson. Five sequential bugs on
one seam — one of which (BUG #4) had to be partly reverted by the next — says the reuse contract
should have been designed once up front. And the milestone re-learned that the local gate is not the
CI gate: `shellcheck -S warning` hid an `info` finding, and the BATS suite drifted until PR CI caught
it. The operator's live `make credential-test PROVIDER=aws` remained the only acceptance that counted.
