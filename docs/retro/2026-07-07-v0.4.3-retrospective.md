# Retrospective — v0.4.3

**Date:** 2026-07-07
**Milestone:** ACG session-check render-race false-negative fix
**PR:** #35 — squash-merged to main (`b7d08b3`); release stamped `5a9bfbe`, tagged `v0.4.3`
**Participants:** Claude, Codex, Copilot

## What Went Well

- **Diagnosed live, not from a hunch.** The false `ACTION REQUIRED: Please log into Pluralsight`
  prompt was reproduced from a real `CLUSTER_PROVIDER=k3s-aws make up` run where the CDP browser
  was already signed in. Probing the six `LOGGED_IN_SELECTORS` moments later showed
  `Cloud Sandboxes`/`Open Sandbox` visible — proving selectors + session were fine and pinning the
  cause to render timing, not auth.
- **Backward-compatible hardening.** `pageLooksLoggedIn` gained an optional
  `{ attempts, perSelectorTimeoutMs, settleMs }` — no options = single-shot exactly as before, so
  no caller was silently changed. `LOGGED_IN_SELECTORS` and the credential/auto-login gating were
  left untouched.
- **Regression proven, not asserted.** Reverting only Change 1 turns the `attempts:4` test red
  while `attempts:1` stays green — the test actually exercises the retry, not just the happy path.
- **Codex flagged the spec gap instead of silently expanding scope.** Change 4 (the
  `acg_session_check.test.js` mock needing `waitForLoadState`) was surfaced mid-task and the spec
  was amended (`92180ee`) rather than quietly touching an unlisted file.

## What Went Wrong

- **The spec's file list missed a second test double.** `_main`'s new `networkidle` settle called
  `page.waitForLoadState`, but only `pluralsight_login.test.js` was listed; `acg_session_check.test.js`
  also drives `_main` and its mock `Page` lacked the method, crashing two existing tests. Caught
  during implementation, but it should have been in the spec.
- **The retry hardening re-exposed a latent O(n) cost.** Adding `attempts` multiplied an already
  *sequential* per-selector timeout in `anyVisible` (6 × 1.5s worst case), so `{ attempts: 4 }` on a
  genuinely logged-out page could balloon to tens of seconds. Copilot caught it; fixed by
  parallelizing `anyVisible` (`487b2f9`) so each attempt is bounded by one per-selector timeout.
- **A stale local `origin/main` ref nearly caused a mis-tag.** During post-merge, the un-fetched
  `origin/main` showed an empty `[Unreleased]` and made the squash-merged commits look absent. A
  `git fetch` before reasoning about merge state would have avoided the detour.

## Process Rules Added

| Rule | Where it applies |
|---|---|
| When a spec adds a call to an optional Playwright `Page` method inside a helper, the spec's file list must include **every** test double that constructs a fake `Page` for that helper's call graph — not just the helper's own test file. | ACG/Playwright specs |
| `.catch()` cannot suppress a **synchronous** `TypeError` from a missing `Page` method; do not add a `typeof` guard around production `Page` calls to satisfy a mock — fix the test double instead (test doubles own the contract). | ACG/Playwright code review |
| When a retry/`attempts` loop wraps a helper, audit whether that helper is itself O(n) sequential — retries multiply the inner cost. | any retry hardening |
| `git fetch` the release remote before reasoning about post-merge state; a squash merge makes branch commits look absent and a stale `origin/main` shows the pre-merge changelog. | post-merge / release |

## Decisions Made

- **`anyVisible` is parallel now** — resolves `true` on the first visible selector, `false` only once
  all resolve. Both callers (`pageLooksLoggedIn` via `LOGGED_IN_SELECTORS`, and MFA detection via
  `MFA_SELECTORS`) keep identical boolean semantics; the fast logged-in happy path is unchanged.
- **Copilot's `.catch` finding resolved by design, not code.** A real Playwright `Page` always
  exposes `waitForLoadState`/`waitForTimeout`; the only failure was a test mock, fixed by Change 4.
  A production `typeof` guard would mask genuine `Page`-contract violations.

## Theme

A single-shot login probe that was correct on a fast render quietly lied on a slow one — the SPA
had not painted when the six selectors were checked, so an already-signed-in session read as logged
out and dropped to a manual prompt. The fix was not new selectors or new auth, but *patience*: retry
across a short settle window, wait for `networkidle`, and stop swallowing navigation failures.
Hardening the timing then re-surfaced an older sin — a sequential per-selector scan whose cost the
single-shot probe had masked — which Copilot caught and we parallelized. The lesson that carries
forward is smaller than the bug: when you add a retry, you multiply whatever is inside it, and when a
helper calls an optional `Page` method, every fake `Page` in its call graph is part of the spec.
