# Retrospective — v0.3.19

**Date:** 2026-05-03
**Milestone:** `_copilot_review` guard-arg fixes + `_copilot_auth_check` gate removal
**PRs:** #25 (`_copilot_auth_check`), #26 (`_copilot_review` deny-tool) — both merged to main (`45040e22`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- Root cause of the deny-tool bug was precise: 4 flags missing closing `)`, caught by running `copilot_draft_spec` manually and tracing the Copilot CLI exit 1 back to the arg parser
- Copilot correctly identified the missing `--allow-all-tools` flag — the deny-tool list is silently ignored without it; the combined fix (PR #26) would have been incomplete without that catch
- Copilot's BATS coverage request was legitimate: a future typo in `--deny-tool` patterns would silently break every `_ai_agent_review` caller; `copilot_review.bats` now guards that
- `copilot_review.bats` stub pattern (override `_safe_path`, `_ensure_copilot_cli`, `_k3dm_repo_root`, `_run_command`) is reusable for any future `_copilot_*` function tests

## What Went Wrong

- **v0.3.18 was a phantom version**: PR #25 was tracked as "v0.3.18 shipped" in memory-bank but the CHANGE.md version was never bumped and no tag was created. Both PR #25 and PR #26 shipped together under v0.3.19, skipping v0.3.18 entirely.
- **Branch tracking misconfiguration**: `fix/copilot-deny-tool-patterns` tracked `origin/main` instead of `origin/fix/copilot-deny-tool-patterns`; push was initially rejected with a branch-protection error. Fixed by patching `branch.<name>.merge` in git config.

## Process Rules Added

| Rule | Rationale |
|------|-----------|
| Every PR that ships code must bump CHANGE.md and create a tag before the next branch is cut | v0.3.18 slipped through without a CHANGE.md entry or tag; caught at v0.3.19 post-merge |

## Decisions Made

- **v0.3.18 skipped**: rather than retroactively create a phantom tag, v0.3.19 incorporates both PR #25 and PR #26 content
- **`copilot_review.bats` stubs `_run_command`**: captures the full argv to a log file; `grep` assertions confirm flag presence without parsing positional structure

## Theme

Two PRs that should have been two separate releases (v0.3.18 and v0.3.19) collapsed into one because the CHANGE.md version bump was skipped after PR #25. The deny-tool fix uncovered a real runtime breakage — every `_ai_agent_review` caller had been silently failing since the flag was introduced. Copilot caught the missing `--allow-all-tools` companion; without it the deny-tool list would have had no effect even with correct patterns.
